    Introduction

    This is the API docs of Sitcom CRM.

    Authentication

    A api_key parameter must be passed to all requests. Api keys are available to administrators in Sitcom.

    Labs

    Get all labs

    curl -X GET https://example.com/api/labs?api_key=XXXX"

    {
  "labs": [
    {
      "id": 2,
      "name": "e-Health",
      "url": "http://sitcom.dev/e-health"
    },
    {
      "id": 3,
      "name": "Fake",
      "url": "http://sitcom.dev/fake"
    },
    {
      "id": 4,
      "name": "fake2",
      "url": "http://sitcom.dev/fake2"
    },
    {
      "id": 1,
      "name": "Smart Gastronomy Lab",
      "url": "http://sitcom.dev/smart-gastronomy-lab"
    }
  ]
}

    This endpoint retrieves all labs.

    HTTP Request

    GET /api/labs

    Query Parameters

    Parameter Type Required
    api_key String Yes

    Get one lab

    curl -X GET https://example.com/api/labs/1?api_key=XXXX"

    {
  "lab": {
    "id": 1,
    "name": "Smart Gastronomy Lab",
    "url": "http://sitcom.dev/smart-gastronomy-lab"
  }
}

    This endpoint retrieves a specific lab.

    HTTP Request

    GET /api/labs/:id

    Query Parameters

    Parameter Type Required
    api_key String Yes

    Create a user

    curl -X POST -H "Content-Type: multipart/form-data;" -F "user[name]=John Doe" -F "user[email]=john.doe@gmail.com" -F "user[password]=superpassword" "http://example.com/api/users?api_key=XXXX"

  "user": {
    "id": 6,
    "name": "John Doe",
    "email": "john.doe@gmail.com",

    "labs": []
  }

    Parameter Type Required
    api_key String Yes
    permission[can_read_contacts] Boolean No

    permission[can_read_organizations ] Boolean No
    permission[can_write_organizations] Boolean No
    permission[can_read_projects] Boolean No
    permission[can_write_projects] Boolean No
    permission[can_read_events] Boolean No
    permission[can_write_events] Boolean No
